    The low-level support

    The technology-dependent part of the robot is embedded in the object nanoSupport.kt that provides two main operations: 
    fun create(owner: ActorBasic, 
	withSonar : Boolean = true)
    		Creates a support to be used by the given owner actor.
    The flag withSonar is used to create a sonarHCSR04Support.kt that transforms the owner in a event stream.

    Ths operation is called by the robotSupport.kt that creates the proper support for each specific robot type specified in the file basicRobotConfig.pl. 
    fun move( cmd : String )
    		Send a move command to the robot by using the motorscSupport.kt that in turn uses Motors.c.

    The robot adapter

    The actor robotAdapterQaStream.kt provides an 'adapter-utility' that allows application designers to send commands the robot actuators (e.g. to execute robot move-commands) without knowing any detail of their low-level configuration.

    Moreover, the adapter works as a an observable data source with respect to the data generated by the robot senors. For example, robot-sonar data can be made available by this component in different forms; for example:
    1. as a stream of values that can be handled by a proper subscriber
    2. as a generator of qak-events, e.g. sonarRobot : sonar( D )
    Since a sonar usually emits data with a quite high frequency and since these data are sometimes not reliable, a good strategy (that we will adopt in the following) is to avoid the direct generation of qak-events by the robotAdapterQaStream.kt (see also LabQakIntro2020.html | Observable Actors).

    Rather, this component could better work as a data-stream source connected with a filter able to select more reliable data and to generate qak-events for those data only.

    Moreover, we can easily add to this data-pipe another filter able to generate other events, e.g. obstacle : obstacle( D ) when the sonar distance is under a prefixed limit.

    Run

    1. Activate the MQTT broker (mosquitto) See LabQakIntro2020.html | Using MQTT
    2. Activate basicrobot.qak on Raspberry
    3. Activate a qak radar radargui.qak in project it.unibo.qak.radar2020
    4. Launch the Jupyter notebook BasicRobotUserCmdMqt.ipynb
    5. Launch a frontend web-server(C:\Didattica2018Work\iss2019Lab\it.unibo.frontend19\nodeCode\frontend) and open a browser on PC-IP:8080
